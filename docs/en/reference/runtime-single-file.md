# Single-file NFZ runtime

This page provides a deliberately simplified comparison view between the runtime generated by **nuxt-feathers-zod** and an `app.js` generated by the official Feathers CLI.

The full file is available in the repository:

```txt
examples/nfz-runtime-single-file.app.ts
```

## Goal

NFZ usually does not start a standalone Node server like a classic Feathers CLI project. It composes a Feathers v5 server inside the Nuxt/Nitro lifecycle, then exposes REST and Socket.IO through generated templates.

The global file is therefore a **mental model**:

- inspect Feathers app creation;
- inspect plugin order;
- compare REST, Socket.IO, local auth, MongoDB and services;
- diagnose differences between an embedded NFZ runtime and a native Feathers `app.js`.

## Mapping with Feathers CLI app.js

| Native Feathers CLI | NFZ embedded |
| --- | --- |
| `src/app.ts` / `app.js` | generated `.nuxt/feathers/server/app.ts` |
| `app.configure(rest())` | embedded REST server template |
| `app.configure(socketio())` | embedded Socket.IO server template |
| `app.configure(authentication)` | `authentication.ts` template when auth is enabled |
| `app.configure(services)` | `feathers.servicesDirs` scan + generated registrations |
| standalone Express/Koa server | Nitro/H3 bridge + Nuxt runtime |

## Recommended usage

This file is not intended to replace NFZ templates. It is a readable runtime snapshot close to Feathers CLI, useful for audits, backports and behavior comparisons.
